# Code Walkthrough: Linear Regression Lab

Этот файл нужен для подготовки к защите. Он объясняет, какие части проекта за что отвечают и как связаны теория, приложение и notebook.

## 1. index.html

`index.html` — обучающая статья для новичка.

Основные блоки:

- шапка сайта с навигацией по разделам;
- hero-блок с названием проекта и кнопкой открытия практики;
- объяснение, что регрессия предсказывает число;
- объяснение данных: `x`, `y`, `ŷ`;
- формула линейной модели `ŷ = a · x + b`;
- residual: `eᵢ = yᵢ − ŷᵢ`;
- RSS и причина, почему ошибки возводятся в квадрат;
- раздел “Откуда берётся формула для a и b?”;
- пошаговый расчёт `Sxx`, `Sxy`, `a`, `b`;
- раздел метрик: RSS, MSE, RMSE, MAE;
- отдельное объяснение R² через baseline-модель “всегда среднее y”;
- объяснение параболы `ŷ = q · x² + a · x + b`;
- ограничения линейной регрессии;
- финальная кнопка “Открыть интерактивную практику”.

Формулы в статье записаны в LaTeX и рендерятся через MathJax, подключённый в `<head>`.

Кнопка перехода в приложение находится в начале страницы и в финальном блоке практики. Обе ведут на `app.html`.

## 2. app.html

`app.html` — интерактивное web-приложение, но оно остаётся обычной статической HTML-страницей.

Основные части:

- `canvas#regressionCanvas`: интерактивный график;
- крупный блок управления под графиком: модель, residuals, датасет и кнопки;
- responsive-сетка `info-grid`: карточки с моделью, метриками, выбранной точкой, прогнозом и объяснением;
- первая колонка: текущая модель и прогноз;
- вторая колонка: метрики;
- третья колонка: выбранная точка;
- ниже на всю ширину: блок “Что сейчас происходит?”;
- переключатель модели: “Линия” или “Парабола”;
- выбор учебного набора: простой, шумный, с выбросом, нелинейный;
- панель выбранной точки с `x`, `y`, `ŷ`, residual и `residual²`;
- input-поля для ручного редактирования координат выбранной точки;
- блок “Что сейчас происходит?” с живым расчётом формул;
- раскрывающийся блок матричного расчёта для параболы;
- блок метрик: RSS, MSE, RMSE, MAE, R²;
- сравнение линейной модели и параболы по MSE и R²;
- поле прогноза: пользователь вводит `x` и получает `ŷ`;
- нижняя мобильная навигация.

Почему больше нет огромных карточек точек:

Раньше каждая точка рисовалась отдельной карточкой с большими кнопками. При большом количестве точек это растягивало страницу и делало интерфейс нечитаемым. Сейчас список точек не показывается отдельным блоком: основной экран показывает количество точек, выбранную точку и основные действия. Чтобы выбрать точку, пользователь нажимает прямо на неё на графике.

MathJax подключён в `index.html` и `app.html` через CDN:

```html
<script defer src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-svg.js"></script>
```

Формулы находятся в блоках `.formula-card` и `.mini-formula`. Они записаны как LaTeX:

```text
\[
R^2 = 1 - \frac{RSS}{TSS}
\]
```

Матричный расчёт для параболы теперь тоже записан через LaTeX. В раскрывающемся блоке показываются:

- матрица признаков `X`;
- вектор ответов `y`;
- произведение `XᵀX`;
- произведение `Xᵀy`;
- итоговый вектор `θ = [b, a, q]`.

## 3. app.js

`app.js` содержит всю логику приложения без библиотек и backend.

### Основные переменные

`points`

Массив всех точек. Каждая точка имеет вид:

```js
{ id: 1, x: 22, y: 3.2 }
```

`selectedPointId`

ID выбранной точки. Если точка не выбрана, значение равно `null`.

`draggingPointId`

ID точки, которую пользователь сейчас перетаскивает мышью или пальцем.

`modelType`

Тип текущей модели:

- `"linear"` для прямой;
- `"quadratic"` для параболы.

`showResiduals`

Булево значение. Если `true`, на графике рисуются вертикальные линии ошибок.

`canvas`

HTML-элемент графика.

`ctx`

2D-контекст canvas. Через него рисуются оси, точки, линия, парабола и residuals.

`mathRenderTimer`

Таймер для debounce-перерисовки MathJax. Он нужен, чтобы при перетаскивании точки приложение не вызывало `MathJax.typesetPromise()` слишком часто.

## 4. Основные функции app.js

Что возвращает: ничего.

Зачем нужна: полностью перерисовывает график. Сначала очищает canvas, затем рисует оси, residuals, модель и точки.

### drawPoints()

Что возвращает: ничего.

Зачем нужна: рисует все точки. Выбранная точка выделяется другим цветом и кольцом.

### drawLineModel(model)

Что принимает: обученную линейную модель.

Что возвращает: ничего.

Зачем нужна: рисует прямую `ŷ = a·x + b`.

### drawQuadraticModel(model)

Что принимает: обученную квадратичную модель.

Что возвращает: ничего.

Зачем нужна: рисует плавную параболу `ŷ = q·x² + a·x + b`.

### drawResiduals(model)

Что принимает: текущую модель.

Что возвращает: ничего.

Зачем нужна: рисует вертикальные линии от реальных точек до предсказания модели. Для прямой линии residual идёт до прямой, для параболы до параболы.

### fitLinearRegression(list)

Что принимает: массив точек.

Что возвращает: объект модели.

Зачем нужна: вручную считает линейную регрессию через:

```text
x̄ = среднее x
ȳ = среднее y
Sxx = Σ(xᵢ - x̄)²
Sxy = Σ(xᵢ - x̄)(yᵢ - ȳ)
a = Sxy / Sxx
b = ȳ - a·x̄
```

Если точек меньше двух или все `x` одинаковые, функция возвращает сообщение об ошибке.

### fitQuadraticRegression(list)

Что принимает: массив точек.

Что возвращает: объект квадратичной модели или объект с сообщением об ошибке.

Зачем нужна: находит коэффициенты параболы

```text
ŷ = q·x² + a·x + b
```

методом наименьших квадратов.

Для квадратичной регрессии матрица признаков и вектор коэффициентов имеют вид:

```text
X = [
  [1, x₁, x₁²],
  [1, x₂, x₂²],
  ...
]

θ = [b, a, q]
```

Минимизация суммы квадратов ошибок приводит к системе нормальных уравнений:

```text
(XᵀX)θ = Xᵀy
```

Если матрица `XᵀX` обратима, математическое решение можно записать формулой:

```text
θ = (XᵀX)⁻¹Xᵀy
```

Эта формула объясняет, какое решение ищется, но код не вычисляет обратную матрицу. Вместо этого `fitQuadraticRegression()` сразу собирает матрицу системы из сумм:

```text
A = XᵀX = [
  [n,    Σx,   Σx²],
  [Σx,   Σx²,  Σx³],
  [Σx²,  Σx³,  Σx⁴]
]

c = Xᵀy = [
  Σy,
  Σxy,
  Σx²y
]
```

После этого функция вызывает:

```js
const solution = solve3x3(matrix, vector);
```

То есть фактически решается система:

```text
Aθ = c
```

где `θ = [b, a, q]`.

Если точек меньше трёх, функция возвращает сообщение: “Для параболы нужно минимум 3 точки”. Если система вырожденная, например из-за недостаточного разнообразия значений `x`, `solve3x3()` возвращает `null`.

### solve3x3(matrix, vector)

Что принимает:

- квадратную матрицу коэффициентов 3×3;
- правую часть системы из трёх чисел.

Что возвращает:

- массив `[b, a, q]`, если система имеет устойчиво определяемое решение;
- `null`, если найден слишком маленький опорный элемент и матрица считается вырожденной.

Зачем нужна: решает систему нормальных уравнений без внешних библиотек.

Функция использует метод Гаусса — Жордана с частичным выбором главного элемента:

1. объединяет матрицу и правую часть в расширенную матрицу;
2. в каждом столбце выбирает строку с наибольшим по модулю опорным элементом;
3. переставляет выбранную строку на текущую позицию;
4. делит опорную строку на опорный элемент;
5. обнуляет текущий столбец во всех остальных строках;
6. считывает решение из последнего столбца расширенной матрицы.

Важное различие:

```text
θ = (XᵀX)⁻¹Xᵀy
```

— это математическая запись решения при существовании обратной матрицы, а `solve3x3()` — конкретный численный способ получить то же решение без явного вычисления `(XᵀX)⁻¹`.

### calculateMetrics(model, list)

Что принимает:

- текущую модель;
- массив точек.

Что возвращает: модель, дополненную метриками.

Зачем нужна: считает:

- предсказания `ŷ`;
- residuals `y - ŷ`;
- `residual²`;
- RSS;
- MSE;
- RMSE;
- MAE;
- TSS;
- R².

R² считается так:

```text
R² = 1 - RSS / TSS
```

Здесь TSS это ошибка baseline-модели “всегда предсказывать среднее y”.

## 5. Как работает перетаскивание

Приложение использует pointer events:

- `pointerdown`: выбирает точку или создаёт новую;
- `pointermove`: двигает выбранную точку;
- `pointerup`: завершает движение.

Это работает и мышью на desktop, и пальцем на телефоне. После каждого движения вызывается `updatePoint()`, а затем `updateAll()`. Поэтому линия, парабола, residuals, метрики и формулы обновляются сразу.
