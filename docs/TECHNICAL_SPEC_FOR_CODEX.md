## Часть 1. Сайт-статья `index.html`

Сделать страницу, похожую по смыслу на статью на Хабре, но проще и понятнее.

Обязательные блоки:

1. Hero-блок:
   - название: “Линейная регрессия без боли”;
   - краткое объяснение;
   - кнопка “Открыть мобильную практику” со ссылкой на `app.html`.

2. Раздел “Что такое линейная регрессия?”:
   - объяснить, что это алгоритм для предсказания числа;
   - привести пример: площадь квартиры → цена квартиры.

3. Раздел “Что такое x, y и ŷ?”:
   - `x` — входной признак;
   - `y` — настоящий ответ;
   - `ŷ` — предсказание модели.

4. Раздел “Формула модели”:
   - показать формулу `ŷ = a · x + b`;
   - объяснить каждую часть:
     - `a` — наклон;
     - `b` — сдвиг;
     - `x` — признак;
     - `ŷ` — предсказание.

5. Раздел “Как модель обучается?”:
   - объяснить, что модель ищет линию с минимальной ошибкой;
   - объяснить residual:
     - `residual = y − ŷ`.

6. Раздел “Как посчитать a и b?”:
   - средние значения:
     - `x̄ = среднее x`;
     - `ȳ = среднее y`;
   - суммы:
     - `S_xy = Σ(xᵢ − x̄)(yᵢ − ȳ)`;
     - `S_xx = Σ(xᵢ − x̄)²`;
   - коэффициенты:
     - `a = S_xy / S_xx`;
     - `b = ȳ − a · x̄`.

7. Раздел “Метрики”:
   - RSS: `RSS = Σ(yᵢ − ŷᵢ)²`;
   - MSE: `MSE = RSS / n`;
   - RMSE: `RMSE = √MSE`;
   - MAE: `MAE = среднее |yᵢ − ŷᵢ|`;
   - R²: `R² = 1 − RSS / TSS`.

8. Раздел “Когда линейная регрессия не подходит?”:
   - нелинейные данные;
   - выбросы;
   - корреляция не равна причинности.

9. Финальный вывод:
   - кратко повторить главные идеи без повторных CTA.

## Часть 2. Мобильное web-приложение `app.html`

Приложение должно выглядеть как мобильное web-приложение, но занимать весь доступный экран. На больших экранах контент может иметь ограничение по ширине для читаемости, но не должен выглядеть как маленький телефонный макет в центре.

### Главные функции

1. Интерактивный график на Canvas.
2. Пользователь нажимает/тапает по графику — добавляется точка.
3. После 2 точек автоматически строится линия линейной регрессии.
4. Показывать axes:
   - X: площадь `x`, м²;
   - Y: цена `y`, млн ₽.
5. Включение/выключение residuals:
   - вертикальные линии от реальной точки до предсказанной точки на линии.
6. Выбор датасета:
   - нормальные квартиры;
   - идеальная линия;
   - шумные данные;
   - выброс;
   - нелинейная зависимость;
   - пустой график.
7. Кнопки:
   - удалить последнюю точку;
   - очистить;
   - вернуть лучшую линию.

### Расчёт модели

В `app.js` реализовать формулы вручную, без библиотек.

```js
meanX = sum(x) / n
meanY = sum(y) / n
S_xy = sum((x_i - meanX) * (y_i - meanY))
S_xx = sum((x_i - meanX) ** 2)
a = S_xy / S_xx
b = meanY - a * meanX
yHat_i = a * x_i + b
residual_i = y_i - yHat_i
RSS = sum(residual_i ** 2)
MSE = RSS / n
RMSE = Math.sqrt(MSE)
MAE = sum(abs(residual_i)) / n
TSS = sum((y_i - meanY) ** 2)
R2 = 1 - RSS / TSS
```

Обработать случаи:
- меньше 2 точек;
- все `x` одинаковые;
- `TSS = 0`.

### Ручной режим

Добавить два range-слайдера:

- Наклон `a`;
- Сдвиг `b`.

Когда пользователь двигает слайдеры:
- включается ручной режим;
- линия строится по выбранным `a` и `b`;
- метрики пересчитываются для ручной линии;
- видно, что плохая линия даёт большие ошибки.

Кнопка “Вернуть лучшую линию” выключает ручной режим и возвращает коэффициенты least squares.

### Формулы с числами

В приложении должен быть блок:

- `x̄ = ...`, `ȳ = ...`;
- `a = S_xy / S_xx = ... / ... = ...`;
- `b = ȳ − a · x̄ = ...`;
- `RSS = Σ(y − ŷ)² = ...`;
- `MSE = RSS / n = ...`.

### Метрики

Показывать карточки:

- количество точек;
- RSS;
- MSE;
- RMSE;
- MAE;
- R².

### Предсказание

Сделать input “Введи площадь x, м²”. Показывать `ŷ = a · x + b` и результат в млн ₽.

### Таблица точек

Показать таблицу:

| # | x | y | ŷ | y − ŷ |

Она должна обновляться при каждом изменении точек или линии.
